### Q3: How do you approach forecasting?

I use a Commit-Expected-Best model:

- Commit: 100% confident will close (verbal/signed)
- Expected: Weighted by stage, rep history, deal health
- Best: Upside scenario

Built this in HubSpot with custom fields. Every week, we knew exactly what we'd close - not a range.

At Penbrothers, this was a key driver of the 3x growth. Leadership trusted the numbers.

## Automation Ideas

### 1. Renewal Risk Detection Dashboard

What we're building: A dashboard that tracks customer engagement metrics (login frequency, support tickets, feature usage) to predict renewal risk.

Why: Renewals are often lost before anyone notices. Sales ops only sees the deal when it's already at risk.

What to expect: Early warning system for at-risk renewals. Identification of expansion opportunities.

Step-by-step:
1. Export customer usage data from platform
2. Build engagement score
3. Create dashboard showing risk levels
4. Set up alerts when score drops

Time: 4-6 hours | ROI: 10-20% improvement in renewals

### 2. CRM Hygiene Automation

What we're building: Automated workflows to keep CRM data clean.

Why: With large team, data decays fast. Duplicates, incomplete records.

What to expect: Weekly duplicate detection. Data completeness scoring.

Step-by-step:
1. Enable duplicate detection
2. Create completeness score property
3. Build workflow for low scores

Time: 3-5 hours | ROI: Cleaner data

### 3. Cross-Region Dashboard

What we're building: Unified view of renewal pipeline across regions.

Why: Different regions = different rhythms. Need unified visibility.

Step-by-step:
1. Create region property
2. Build dashboard with filters
3. Add forecast roll-ups

Time: 4-6 hours | ROI: Real-time visibility
